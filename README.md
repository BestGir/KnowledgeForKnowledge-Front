# KnowledgeForKnowledge Frontend

Фронтенд платформы обмена знаниями `KnowledgeForKnowledge`.
Проект реализован на `React + TypeScript + Vite` и работает с backend API из основного репозитория.

## Что делает приложение

- регистрация, вход, восстановление пароля
- подтверждение входа через Telegram OTP
- каталог предложений и запросов с фильтрами, поиском и пагинацией
- отклики на карточки и управление своим потоком откликов
- личный кабинет: профиль, фото, навыки, пруфы, образование
- публичные профили пользователей с отзывами
- админ-панель: модерация пруфов, удаление карточек, управление каталогом навыков

## Технологии

- React 19
- TypeScript 6
- Vite 8
- React Router 7

## Требования

- Node.js 20+
- npm 10+
- запущенный backend API (по умолчанию `http://localhost:5000`)

## Быстрый старт

1. Установи зависимости:

```bash
npm install
```

2. Создай `.env` на основе `.env.example` (если нужен кастомный адрес API).

3. Запусти dev-сервер:

```bash
npm run dev
```

4. Открой приложение в браузере (обычно `http://127.0.0.1:5173`).

## Переменные окружения

`.env.example`:

```bash
VITE_API_BASE_URL=http://localhost:5000/api
```

Если `VITE_API_BASE_URL` не задан, фронт использует относительные пути `/api`.

## Взаимодействие с API

- В dev-режиме `vite` проксирует:
- `/api` -> `http://localhost:5000`
- `/uploads` -> `http://localhost:5000`

Это позволяет работать без CORS-настройки в браузере при локальной разработке.

## Команды

```bash
npm run dev      # локальная разработка
npm run build    # production-сборка
npm run preview  # просмотр production-сборки
npm run lint     # линтер
```

## Структура проекта

- `src/pages` — страницы приложения
- `src/components` — переиспользуемые UI-компоненты
- `src/auth` — контекст и логика сессии
- `src/lib` — API-клиент, форматирование, утилиты
- `public` — статические файлы

## Что стоит проверить вручную после изменений

- вход/регистрация/OTP
- создание и удаление карточек в каталоге
- отклики и статусы сделок
- редактирование профиля, навыков, пруфов и образования
- админские сценарии (модерация и удаление карточек)
